# MVP 技术方案

> 第一版目标：**5 秒钟生成一个可以发给客户的 HTML。**

---

## 技术选型

选用 **Electron**，原因：
- Vue ✅ 已掌握
- Node.js ✅ 已掌握
- Three.js ✅ 已掌握
- 只需学习 Electron API 即可起步
- 目标是把产品做出来，不是做技术展示

---

## 第一版架构

```
Electron
│
├── Vue（界面层）
│
├── Main（Electron 主进程）
│   ├── 文件选择
│   ├── 保存 HTML
│   ├── 导出
│   └── 自动打开文件夹
│
├── Renderer（渲染进程）
│   ├── 拖拽上传
│   ├── 参数设置
│   └── 预览
│
└── Builder（构建引擎）
    ├── 读取 GLB
    ├── Base64 编码
    ├── 读取贴图
    ├── 注入模板
    └── 输出 demo.html
```

## 目录结构

按产品级标准组织，为后续扩展预留空间：

```
3d-presenter/
├── electron/              # Electron 主进程
│   ├── main.ts
│   └── preload.ts
│
├── src/
│   ├── components/        # Vue 组件
│   ├── pages/             # 页面
│   │
│   ├── builder/           # 构建引擎（核心）
│   │   ├── htmlBuilder.ts     # HTML 组装与注入
│   │   ├── glbReader.ts       # GLB 解析
│   │   └── assetEncoder.ts    # 资源 Base64 编码
│   │
│   ├── template/          # HTML 模板
│   │   ├── viewer-dark.html       # 模板A：深色
│   │   ├── viewer-light.html      # 模板B：白色
│   │   ├── viewer-product.html    # 模板C：产品展示
│   │   ├── viewer-industrial.html # 模板D：工业
│   │   └── viewer-tech.html       # 模板E：科技蓝
│   │
│   └── utils/
│
├── resources/             # 静态资源
│   ├── hdr/
│   └── icons/
│
└── output/                # 导出产物
```

## 模板化设计（从第一版就预留）

HTML 模板不写死，设计为可切换：

| 模板 | 代号 | 适用场景 |
|------|------|----------|
| 深色 | `dark` | 通用 / 默认 |
| 白色 | `light` | 建筑 / 室内 |
| 产品展示 | `product` | 电商 / 消费品 |
| 工业 | `industrial` | 工厂 / 设备 |
| 科技蓝 | `tech` | 数字孪生 / 数据可视化 |

模板本身即可成为收费点，从第一版就预留好切换机制，避免后期重构。
