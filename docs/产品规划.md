# 产品规划：3D 离线交付平台

> **给 Claude 的前置指令：不要把它设计成一个"工具"，而是按照未来可以商业化 3~5 年的 SaaS 产品来规划。优先考虑产品架构、可扩展性、插件化、模板化和企业级能力，而不是仅完成当前功能。**

---

## 项目定位

目标不是开发一个简单的 GLB 转 HTML 工具，而是打造一个面向 Three.js、数字孪生、BIM、工业可视化、产品展示等行业的 **3D 离线交付平台**。

核心理念只有一句话：

> **任何一个三维模型，都应该可以一键生成一个无需服务器、无需安装、无需部署、双击即可打开的高质量 HTML 演示文件。**

它不是格式转换工具，而是一个 **3D Presentation Generator（3D 演示生成器）**。

---

## 要解决的问题

目前交付一个 Three.js 项目存在很多麻烦：

- 需要部署服务器
- 需要配置静态资源
- 需要发送多个文件
- 模型、贴图容易丢失
- 路径容易出错
- 客户不会启动项目
- 演示环境不统一
- 每个项目都要重复搭建相机、灯光、环境

**本产品希望把这些问题全部自动化。**

---

## 产品目标

用户上传一个模型 → 经过自动处理后 → 得到一个可以直接交付客户的 HTML 文件。

这个 HTML 不只是能够显示模型，而是已经拥有完整的展示能力：

- 自动灯光
- 自动 HDR
- 自动阴影
- 自动曝光
- 自动相机位置
- 自动模型居中
- 自动动画播放
- 自动旋转
- 自动手机适配
- 自动全屏
- 自动截图
- 自动生成缩略图

**最终生成的不是一个模型，而是一个完整的三维演示页面。**

## 产品路线图

### 第一阶段：核心导出能力

打造业内最好用的 **GLB → 单文件 HTML 导出器**。

重点：
- 所有资源内嵌（模型、贴图、HDR、字体）为单个 HTML
- 自动化场景搭建（灯光、相机、阴影、曝光）
- 响应式布局与移动端适配
- 可配置的 UI 控件（进度条、Logo、水印）
- 输出文件体积优化（Draco、KTX2、量化）

### 第二阶段：多格式支持

扩展输入格式：

- FBX、OBJ、STL、PLY
- IFC（BIM 行业）
- GeoJSON / 3D Tiles（GIS / 数字孪生）
- USD / USDZ（Apple 生态）
- glTF 2.0 全特性（动画、Morph Target、Skinning）

### 第三阶段：可视化编辑

用户无需写代码即可配置：

- 标签与注记
- 热点（Hotspot）
- 点击事件与信息弹窗
- 巡检路线
- 相机动画（关键帧）
- 爆炸图
- 剖切面
- 测量工具（距离、角度、面积）
- 天气效果（雨、雪、雾、昼夜）
- 时间轴（施工进度、装配序列）

### 第四阶段：AI 智能增强

AI 自动分析模型，自动生成：

- 最佳相机视角
- 最佳灯光布局
- 热点位置与标注
- 模型介绍文案
- 演示脚本
- 产品说明
- 自动配色方案
- 演讲旁白脚本

### 第五阶段：完整 3D Presentation Platform

不局限于 HTML 导出：

- 导出 EXE（Electron / Tauri 打包）
- 导出桌面程序（Windows / macOS / Linux）
- 一键发布到云端
- 自动生成分享链接
- 自动生成二维码
- 自动生成产品介绍页
- 团队协作与版本管理

## 产品架构原则

以下是指导整个产品演进的核心架构原则：

### 1. 插件化

- 每种输入格式、输出目标、UI 组件、后处理效果都以插件形式注册
- 核心引擎保持轻量，功能通过插件扩展
- 第三方开发者可以编写自定义插件

### 2. 模板化

- 演示模板系统：工业、建筑、产品、教育等场景模板
- 模板包含：灯光预设、相机行为、UI 布局、配色方案
- 用户可创建、分享、交易模板

### 3. 管线化

- 模型处理管线：导入 → 解析 → 优化 → 场景搭建 → 演示配置 → 导出
- 每个阶段可独立配置、替换、扩展
- 支持命令行和 API 调用，可集成到 CI/CD

### 4. 企业级能力

- 多租户架构
- RBAC 权限管理
- 批量处理与队列调度
- 审计日志
- SSO / SAML 集成
- 私有化部署
- API 接口与 Webhook

### 5. 离线优先

- 所有导出产物完全自包含，零外部依赖
- HTML 产物可在任何现代浏览器中打开
- EXE 产物不依赖网络

## 产品原则

产品的价值不是"把 GLB 变成 HTML"。

**真正的价值是：让任何人都可以在一分钟内，把一个三维模型变成一个可以直接交付、直接展示、直接分享的专业演示作品。**

整个产品始终围绕三个目标持续演进：

1. **降低交付成本** — 消除服务器、部署、配置等一切非核心工作
2. **提升展示效果** — 自动化最佳实践，让每个模型都能以最佳状态呈现
3. **减少重复开发** — 模板化、插件化，一次配置永久复用

## 技术选型方向（待细化）

- **运行时**: Three.js（WebGL / WebGPU）
- **打包格式**: 单文件 HTML + Base64/Gzip 内嵌资源
- **压缩**: Draco（几何）、KTX2（纹理）、Meshopt
- **编辑器**: 渐进式引入，从表单配置到可视化编辑
- **桌面端**: Tauri（轻量）或 Electron（生态丰富）
- **后端**: 按需引入，初期纯客户端，后期云服务
